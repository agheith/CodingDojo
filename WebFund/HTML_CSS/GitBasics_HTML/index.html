What is Git?

Git is a version control system; you can think of it as a series of snapshots (commits) of your code base. With the creation of each snapshot you build out a whole application, seeing the growth of a full app. During this time you can make branches to experiemnt and come back to a previous snapshots you took or merge those experiments into the your app.

What is GitHub?

GitHub us a code hosting platform for version control and collaboration. It lets you and others work together on projects from anywhere using those code snapshots that you took during development.

Git init

Git init is the act of registerating this current dode base.

Git add

Git add is how we stage the integration of our current code base into our current list of snapshots.

Git status

Git status allows us to see the changes we are going to make.

Git Commit

Git Commit; finall we can commit those changes we added to our last snapshot giving us a new snapshot of our code with whatever new changes were made.

Git remote add

Git remote add is used to set up an online repository where we can place all of our current code. This is where the outside world can see our code.

Git push

Git push is how we get our code to GitHub, an excellent place to store all of our repositories. Other developers might look at our existing projects to see how we approached porblems, pull down our code and add new features or even update our code base with features they wanted to add.
